Cost Pareto Lines

Pareto cost lines show how much sustained QPS each cloud vector database can deliver for a given operating spend. We build these lines from actual search results shown in CloudPayloadSearchCase & MultitenantSearchCase, then apply public pricing models so the chart compares operating efficiency, not just raw benchmark speed.

Notes:

All tests are conducted in AWS us-west-2. All costs shown here are based on each product's charges in this region.

  • Zilliz Cloud tiered is charged at $0.372 USD/CU-hour, Zilliz Cloud capacity is charged at $0.248 USD/CU-hour, and storage uses the AWS us-west-2 list price from Zilliz Cloud pricing.
  • Turbopuffer (unpinned) queries and writes are charged as FP32 usage with a namespace of approximately 302 GB, vector data only, and apply Turbopuffer's official pricing rules.
  • Turbopuffer (pinned) costs use 2 replicas, where utilization falls below 90% as recommended by Turbopuffer's namespace pinning documentation.
  • Pinecone costs use a namespace of approximately 302 GB, vector data only, and apply AWS us-west-2 read, write, and storage unit charges from Pinecone's official pricing rules.
  • Write-inclusive mode models the same monthly corpus size as either constant writes, where LAION 100M maps to 4 write requests/second, or batched writes with 10k records/request.
